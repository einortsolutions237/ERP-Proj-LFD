# Phase 8.1 — Live Verification

Not new feature work. Phase 8 fixed the branch-targeting bug in staff and
department creation and did a systematic review of every branch-scoped
route, but left two gaps: no durable, inspectable record that the live
exercise actually ran (the evidence lived only in a chat transcript and a
now-deleted local script), and no completed per-route review — several
routes were confirmed correct in the earlier phase but never got their own
documented line. This document closes both gaps.

All actions below were performed through the running app: real HTTP
requests against real Next.js API routes, authenticated as real Firebase
accounts via session cookies obtained the same way the browser does
(`POST /api/auth/session` with a real ID token). The controller signed in as
each account using an Admin-SDK custom token for that account's real UID —
this requires only the same service-account trust already used throughout
this project's Admin SDK access, never a stored or guessed password, and
produces a real Firebase ID token indistinguishable from a normal
password-based sign-in once exchanged for a session cookie. No direct
Firestore writes were used for any of the actions below; two verification
reads (confirming Phase 8's Ikeja branch/staff/sale/leave-request still
existed, and checking a leave-approval queue's Firestore state before
approval) used direct Firestore reads only, never writes.

**Branch reuse decision, stated up front:** Phase 8 already created a real,
permanent second branch ("LFD Services — Ikeja Branch") with a
`branch_manager` and `cashier`. Rather than inventing a third branch, this
phase reused Ikeja as "the new branch" and instead closed the actual gap
Phase 8 left: the *original* branch ("LFD Services — Downtown Branch") had
no `branch_manager` of its own, so cross-branch isolation had only ever been
demonstrated in one direction. This phase created a `branch_manager` for
Downtown specifically to make the isolation check bidirectional, and ran
fresh restock/sale/leave-request actions at both branches so this
document's evidence is freshly observed in this session, not carried over
from memory of Phase 8.
